# ナレッジベース (knowledge base)

このディレクトリには、エージェントが回答時に参照するナレッジ（知識）を Markdown
ファイルとして配置します。エージェントは起動時にこのディレクトリ配下の
`.md` / `.markdown` / `.txt` ファイルを読み込み、langchaingo の
`textsplitter` でチャンクに分割してメモリ上に保持します。

ユーザーからの質問に答えるとき、エージェントは `knowledge_search` ツールを
通じて関連するチャンクを検索し、その内容を根拠として回答を組み立てます
（いわゆる Retrieval-Augmented Generation / 検索拡張生成）。

## ナレッジの追加方法

1. このディレクトリ（デフォルトは `knowledge/`）に Markdown ファイルを追加します。
2. 1ファイル1トピックを意識し、見出し（`#`）で内容を構造化すると検索精度が上がります。
3. アプリを再起動すると、新しいナレッジが自動的に読み込まれます。

読み込むディレクトリは環境変数 `AGENT_KNOWLEDGE_DIR` で変更できます。
ディレクトリが存在しない、または空の場合は `knowledge_search` ツールは
登録されず、エージェントは従来どおり動作します。
